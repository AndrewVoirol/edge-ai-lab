# Roadmap

This roadmap provides visibility into where Edge AI Lab is heading. It's organized into three tiers — what we're working on now, what's coming next, and what we're thinking about for the future.

> **Want to influence the roadmap?** Open a [feature request](https://github.com/AndrewVoirol/edge-ai-lab/issues/new?template=feature_request.yml) or join the discussion in [Ideas](https://github.com/AndrewVoirol/edge-ai-lab/discussions/categories/ideas). Contributions to items in **Now** and **Next** are especially welcome.

---

## Now — v2.0 (Active)

*What we're actively building and stabilizing.*

- [ ] v2.0 release stabilization
- [ ] URL import pipeline refinements (error recovery, retry logic)
- [ ] Kaggle import hardening
- [ ] Community model catalog expansion
- [ ] Contributor onboarding improvements (tutorials, good first issues)

## Next — v2.1 (Planned)

*What's coming after the v2.0 release. These are confirmed directions, not commitments to specific timelines.*

- [ ] Additional built-in evaluation suites (contributor-driven)
- [ ] New built-in tools beyond the current 6 (contributor-driven)
- [ ] Performance optimizations for 12B Dense model inference
- [ ] Enhanced MCP server discovery and configuration UI
- [ ] Conversation export formats (Markdown, PDF)
- [ ] Code coverage reporting and thresholds in CI

## Later — Future (Aspirational)

*Ideas we're excited about but haven't committed to yet. Community interest and contributions will shape priorities.*

- [ ] Multi-turn agent workflows with persistent state
- [ ] Model comparison view (side-by-side inference)
- [ ] Plugin/extension architecture for custom tools
- [ ] Signed `.app` distribution (DMG, Homebrew cask)
- [ ] Model fine-tuning support
- [ ] Prompt template library
- [ ] Localization (i18n)
