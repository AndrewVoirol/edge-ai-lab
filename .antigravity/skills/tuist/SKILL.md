# Tuist Project Management

## Core Principle
`Project.swift` is the **single source of truth** for the Xcode project. All generated artifacts (`.xcodeproj`, `.xcworkspace`, `Derived/`) are ephemeral and regenerated by `tuist generate`.

## Critical Rules

> [!CAUTION]
> **NEVER** directly edit `.xcodeproj`, `.xcworkspace`, or anything in `Derived/`. These are Tuist-generated and will be overwritten.

- ALL project changes go through `Project.swift`
- After ANY edit to `Project.swift`, run `tuist generate`
- New Swift files added to `Sources/` or `Tests/` require `tuist generate` to be picked up
- The `.xcworkspace` file has a `.tuist-generated` marker — respect it

## Workflow
1. Edit `Project.swift` (targets, deps, settings, Info.plist)
2. Run `tuist generate` (or use the script: `.antigravity/skills/tuist/scripts/generate.sh`)
3. Build using XcodeBuildMCP tools

## What Goes in Project.swift
- Target definitions (name, platform, product type)
- Bundle identifiers
- Deployment targets
- Info.plist extensions
- Dependencies (packages, targets)
- Build settings (code signing, team ID)
- Source/test file globs

### InfoPlist Keys (iOS Target)
The following keys are configured in the iOS target's `InfoPlist` in `Project.swift`:

| Key | Value | Purpose |
|---|---|---|
| `UIFileSharingEnabled` | `true` | Exposes Documents directory in Finder/Files app for model provisioning |
| `LSSupportsOpeningDocumentsInPlace` | `true` | Allows opening documents directly from Files app |
| `UISupportsDocumentBrowser` | `true` | Enables document browser for model file management |

These keys enable users (and `devicectl`) to provision `.litertlm` model files to the app's Documents directory on physical devices.
